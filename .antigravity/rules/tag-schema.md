# Tag Schema

Tags are controlled discovery facets. They are not a substitute for structured metadata.

## Approved Discovery Tags

- `beginner`
- `advanced`
- `comparison`
- `case-study`
- `implementation`
- `reference`
- `history`
- `decision`
- `example`
- `checklist`
- `open-question`
- `contrarian`

## Rules

- Tags must be lowercase and hyphenated.
- Never invent ad hoc tags. Every tag used in the vault must exist in this file.
- Use zero or more approved discovery tags; do not tag by habit.
