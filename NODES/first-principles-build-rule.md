# First Principles Build Rule

Build from first principles. Never assume existing solutions are optimal. Deconstruct the problem to its fundamental truths and reconstruct from there.

## Explanation

This is the practical application rule derived from [[first-principles-thinking]]. Whereas the thinking method is about how to approach problems, this rule states that active deconstruction and reconstruction should be the default engineering practice, not a special mode used only when stuck.

The rule has three operational steps:
1. Identify the current solution and its cost
2. Deconstruct to fundamental physics and material costs
3. Rebuild from fundamentals with no legacy constraints

## Why it matters

Without this rule, teams default to "reasoning by analogy" — copying existing solutions with incremental changes. That produces convergent design. This rule forces divergent, breakthrough thinking as standard operating procedure.
