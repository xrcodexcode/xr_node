# Git Version Control System

## Definition
Git is a distributed version control system that tracks modifications to project files, allowing developers to create snapshots (commits), revert files to previous states, and collaborate concurrently on multiple feature branches.

## Explanation
- **Repository (Repo)**: A directory initialized and tracked by Git (via the hidden `.git/` folder).
- **Staging Area**: A conceptual middle ground where files are selected and registered (`git add`) before they are saved.
- **Commit**: A permanent local snapshot of staged modifications, capturing what changed, who changed it, and a descriptive reason.
- **GitHub**: A cloud hosting service for Git repositories, allowing remote collaboration via pushes (uploading commits) and pulls (downloading remote revisions).
